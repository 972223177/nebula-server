# Nebula Server — 代码生成规范

## 中文注释规范

所有由 AI 生成的 Kotlin 代码必须遵守以下注释规范：

### 1. 类/接口/枚举

必须添加 `/** ... */` KDoc 注释，说明：
- 该类型的职责和用途
- 涉及的关键设计决策（如有）
- @param 说明构造参数（如果适用）

### 2. 方法/函数

必须添加 `/** ... */` KDoc 注释，说明：
- 方法的功能
- @param 参数说明
- @return 返回值说明（如果适用）
- 关键实现细节（如线程安全策略、边界条件）

### 3. 字段/属性

- data class 构造函数参数：使用 `/** ... */` 注释
- 类内部字段：使用 `/** ... */` 或 `//` 注释
- 注释说明字段的用途和约束（如取值范围、默认值）

### 4. 内联逻辑

- 复杂逻辑块前添加 `// 中文注释` 说明意图
- 分支条件、边界处理等关键节点需要注释

### 5. 注释语言

- **所有文档注释使用中文**
- 技术标识符（package、class名、方法名、关键字）保留英文原文
- 代码中已有的英文注释可保留或替换为中文

### 6. 注释原则

- **注释解释"为什么这么做"**，而非"做了什么"（代码本身已表达功能）
- 不要添加冗余注释（如 `// 返回结果` 在 return 语句前）
- 设计决策引用 `D-编号`（来自 .planning/ 设计决策）以提高可追溯性
