# /nx-status — 项目状态

## 目的

快速查看当前项目的全景状态，包括当前阶段、完成进度、阻塞项和下一步建议。
类似 GSD `gsd-health` + `gsd-progress` 的功能合并。

## 流程

### 第一步：读取状态文件

1. 读取 `.planning/STATE.md` — 项目状态快照
2. 读取 `.planning/PROJECT.md` — 项目背景和需求
3. 读取 `.planning/ROADMAP.md` — 全部阶段概览

### 第二步：探测当前阶段详情

识别"当前阶段"（从 STATE.md），然后：
1. 读取 `.planning/phases/<N>/PLAN.md`（如果存在）
2. 统计任务完成情况

### 第三步：输出状态报告

显示丰富的项目全景：

```markdown
## ═══ 项目状态 ═══

### 项目
[项目名] — [核心价值]

### 当前阶段：阶段 <N>：[名称]
状态：[未开始/规划中/执行中/验证中/已完成]
进度：[X/Y] 任务 → [XX%] 
Wave 进度：Wave 1 [X/Y] | Wave 2 [X/Y] | Wave 3 [X/Y]

### 完成阶段
- 阶段 1：[名称] ✅（完成日期）
...

### 阻塞项
- [如果有未通过验证的任务]
- [如果有未完成的 gap_closure]

### 下一步建议
${根据当前状态推荐下一步命令}
```

### 状态到建议映射

| 状态 | 建议 |
|---|---|
| 无 `.planning/` | `/nx-init` 初始化项目 |
| 已初始化但无规划 | `/nx-plan 1` 规划第一阶段 |
| 有 PLAN 但未执行 | `/nx-exec <N>` 开始执行 |
| 有执行但未验证 | `/nx-verify <N>` 验证 |
| 验证发现问题 | `/nx-exec <N> --gaps-only` 修复 |
| 验证通过 | `/nx-done <N>` 完成 |
| 全部完成 | 提示进入下一阶段或新里程碑 |

### 第四步：快捷建议

在报告最后提供 1-3 个具体命令建议，用户可直接复制执行。
