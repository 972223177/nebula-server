# .planning/ 文档格式规范

本文档定义星云工作流 Plugin 使用的所有文档格式。
完全兼容 GSD 生成的 `.planning/` 文档。

## 目录结构

```
.planning/
├── PROJECT.md            # 项目背景文档（必需）
├── REQUIREMENTS.md       # 需求文档（必需）
├── ROADMAP.md            # 路线图文档（必需）
├── STATE.md              # 项目状态文档（必需）
├── config.json           # 工作流配置（必需）
├── research/             # 技术研究资料（可选）
│   └── *.md
├── phases/               # 阶段目录
│   ├── 1/
│   │   ├── CONTEXT.md    # 讨论上下文（可选）
│   │   ├── RESEARCH.md   # 技术研究（可选）
│   │   └── PLAN.md       # 执行计划（必需）
│   ├── 2/
│   │   └── ...
│   └── ...
├── quick/                # 快捷记录（可选）
└── intel/                # 代码库分析记录（可选）
```

---

## PROJECT.md

项目背景文档，描述项目的整体目标和约束。

```markdown
# [项目名称]

## What This Is

[2-3 句话描述产品做什么、给谁用]

## Core Value

[一句话核心价值 — 如果其他都失败了，这个必须成功]

## Requirements

### Verified（已验证）

<!-- 已交付并验证的需求 -->

- ✓ [需求] — [阶段/版本引用]

### Active（活跃）

<!-- 当前范围，正在构建 -->

- [ ] [需求 1]
- [ ] [需求 2]

### Out of Scope（范围外）

- [排除项 1] — [原因]
- [排除项 2] — [原因]

## Context

[背景信息：技术环境、相关工作、用户反馈]

## Constraints

- **[类型]**：[内容] — [原因]
- **[类型]**：[内容] — [原因]

类型包括：技术栈、时间线、预算、依赖、兼容性、性能、安全

## Key Decisions

| 决策 | 理由 | 结果 |
|------|------|------|
| [选择] | [为什么] | [✓ 好 / ⚠️ 需回顾 / — 待定] |

---

*最后更新：[日期] 在 [触发条件] 之后*
```

## REQUIREMENTS.md

需求文档，列出已验证、活跃和范围外的需求。

```markdown
# 需求文档

## 已验证

<!-- 已交付并验证的需求列表 -->

- ✓ [需求 1] — [实现于阶段 X]
- ✓ [需求 2] — [实现于阶段 Y]

## 活跃

<!-- 当前正在构建或待构建 -->

- [ ] [需求 3] — [描述]
- [ ] [需求 4] — [描述]

## 范围外

<!-- 明确的边界 -->

- [排除项] — [原因]
```

## ROADMAP.md

路线图文档，定义阶段划分。

```markdown
# 路线图

## 概览

| 阶段 | 名称 | 复杂度 | 状态 |
|------|------|--------|------|
| 1 | [名称] | S/M/L | 已完成/进行中/待开始 |
| 2 | [名称] | S/M/L | 待开始 |

## 阶段 1：[名称]

**范围**：[做什么]
**交付物**：[具体产出]
**预估复杂度**：S/M/L
**依赖**：[前置条件]

### 需求映射
- [需求 1]、[需求 2]

---

## 阶段 2：[名称]
...
```

## STATE.md

项目状态文档，由命令自动维护。

```markdown
# 项目状态

## Project Reference

参考：.planning/PROJECT.md（更新于 [日期]）

**核心价值**：[一句话]
**当前焦点**：[当前阶段名]

## 当前阶段

阶段：[号]
名称：[名称]
状态：待开始 | 规划中 | 执行中 | 验证中 | 已完成
进度：[X/Y] 任务完成
验证：[未验证 | 已通过 | 有问题]

## 已完成阶段

- 阶段 1：[名称] ✅（完成日期）

## 阻塞项

- [如果有未解决的问题]
```

## PLAN.md（阶段执行计划）

阶段执行计划，包含任务分解、依赖分析、wave 分组。

```markdown
# 阶段 [N] 执行计划

## 阶段信息

**路线图引用**：[来自 ROADMAP.md]
**复杂度**：S/M/L
**状态**：待开始 | 执行中 | 已完成
**预估工作量**：[人天]

## 任务分解

### Wave 1：基础设施
<!-- 无依赖任务，可并行 -->

#### 任务 1.1：[任务名]
**状态**：待开始 | 执行中 | 已完成 | 失败
**前置**：无
**描述**：[需要做什么]
**验收标准**：
- [ ] [标准 1]
- [ ] [标准 2]
**关联文件**：[文件路径]
**复杂度**：S/M/L
**结果**：[执行完成后记录]

### Wave 2：核心逻辑

#### 任务 2.1：[任务名]
**状态**：待开始
**前置**：任务 1.1
**描述**：[需要做什么]

### Wave 3：集成与验证

## WAVE 配置

waves:
  1:
    tasks: [1.1, 1.2]
  2:
    tasks: [2.1, 2.2]
  3:
    tasks: [3.1]

## 依赖图

```
任务 1.1 ──→ 任务 2.1 ──→ 任务 3.1
任务 1.2 ──→ 任务 2.2
```

## Gap Closure（修复计划）

#### 任务 1.G1：[修复描述]
**状态**：待开始
**gap_closure**: true
**描述**：[修复验证发现的问题]
```

## CONTEXT.md（讨论上下文）

讨论阶段的输出文档，为规划提供输入。

```markdown
# 阶段 [N] 讨论上下文

## 阶段范围

[来自 ROADMAP.md 的阶段范围描述]

## Locked Decisions（锁定决策）

| 决策 | 选择 | 理由 |
|------|------|------|
| [决策名] | [选择] | [理由] |

## 用户偏好

- [偏好 1]
- [偏好 2]

## 排除项

- [明确排除的内容和原因]

## 注意事项

- [特殊提醒]
```
