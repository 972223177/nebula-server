# 偏差处理规则

Executor 在执行 PLAN.md 时遇到偏差的处理策略。

## 偏差类型

| 偏差类型 | 定义 | 处理策略 | 自动修复限制 | 示例 |
|---------|------|---------|-------------|------|
| Bug 修复 | 实现中发现的代码错误 | 自动修复 + 记录到 SUMMARY.md | 无限 | 空指针、类型错误 |
| 关键功能补充 | PLAN 中遗漏的必要功能 | 自动补充 + 记录 | 3 次 | 缺少错误处理 |
| 阻塞修复 | 阻止继续执行的问题 | 尝试 3 次 → 升级给用户 | 3 次 | 依赖缺失 |
| 架构变更 | 涉及架构层面修改 | 立即升级给用户 | 0 次 | 改变 API 契约 |
| 设计偏差 | 与设计预期不符 | 尝试 2 次匹配 → 升级 | 2 次 | UI 与设计稿不符 |
| 兼容性修补 | 兼容性问题 | 自动使用兼容方案 + 记录 | 3 次 | API 版本兼容 |

## 偏差记录格式

在 SUMMARY.md 的 Deviations 部分：

```markdown
## 偏差记录

| # | 类型 | 计划任务 | 描述 | 处理方式 |
|---|------|---------|------|---------|
| 1 | Bug 修复 | Task 2 | 空指针异常 | 添加 null 检查 |
| 2 | 关键功能补充 | Task 3 | 缺少异常处理 | 添加 try-catch + 错误响应 |

如果无偏差：
## 偏差记录
None
```

## 升级流程

当偏差超出自动修复限制时：
1. 暂停当前执行
2. 展示偏差详情和建议方案
3. 用户选择：接受建议 / 修改方案 / 跳过 / 中止
4. 根据选择继续执行

## 分析卡死保护

| 条件 | 行为 |
|------|------|
| 连续 5 次只读工具调用 | 立即停止，要求用户确认方向 |
| 连续 3 次工具调用返回空结果/错误 | 立即停止（无意义搜索） |

触发时输出：
```markdown
⚠️ 分析卡死警告
连续 N 次 <只读调用/空结果>，可能进入无效循环。

最后操作：<描述>
建议：确认 <方向> 是否正确，或切换到 <备选方案>。

[继续 / 切换方向 / 停止]
```
