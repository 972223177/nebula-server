    **创建一个单一 V1 文件**（非 6 个独立文件），按以下顺序建表：

    1. **users** — 用户表
       - id BIGINT NOT NULL PRIMARY KEY（Snowflake ID）
       - username VARCHAR(64) NOT NULL
       - password_hash VARCHAR(128) NOT NULL
       - nickname VARCHAR(64) NOT NULL
       - avatar VARCHAR(256) NOT NULL DEFAULT ''
       - privacy_status TINYINT NOT NULL DEFAULT 0
       - created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
       - UNIQUE KEY uk_username (username)

    2. **conversations** — 会话表（先建，FK 依赖方后建）
       - id VARCHAR(32) NOT NULL PRIMARY KEY（UUID）
       - type TINYINT NOT NULL（1=私聊, 2=群聊）
       - name VARCHAR(128) NOT NULL DEFAULT ''
       - avatar VARCHAR(256) NOT NULL DEFAULT ''
       - group_owner_uid BIGINT DEFAULT NULL（群聊时指向 users.id）
       - member_count INT NOT NULL DEFAULT 0
       - max_members INT NOT NULL DEFAULT 200
       - created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
       - INDEX idx_group_owner (group_owner_uid)

    3. **conversation_members** — 会话成员表
       - id BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY
       - conversation_id VARCHAR(32) NOT NULL
       - user_id BIGINT NOT NULL
       - joined_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - last_read_message_id BIGINT NOT NULL DEFAULT 0
       - unread_count INT NOT NULL DEFAULT 0
       - deleted TINYINT NOT NULL DEFAULT 0
       - UNIQUE KEY uk_member (conversation_id, user_id)
       - INDEX idx_user_convs (user_id)

    4. **messages** — 消息表
       - id BIGINT NOT NULL PRIMARY KEY（Snowflake ID）
       - conversation_id VARCHAR(32) NOT NULL
       - sender_uid BIGINT NOT NULL
       - message_type TINYINT NOT NULL（ChatContentType 枚举值）
       - content TEXT NOT NULL
       - payload BLOB DEFAULT NULL
       - client_message_id VARCHAR(64) DEFAULT NULL（唯一约束，允许一个 NULL）
       - client_ts BIGINT NOT NULL
       - server_ts BIGINT NOT NULL
       - created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - INDEX idx_conv_messages (conversation_id, id) — 游标分页核心索引
       - UNIQUE KEY uk_client_msg_id (client_message_id)

    5. **friendships** — 好友关系表
       - id BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY
       - user_id BIGINT NOT NULL
       - friend_id BIGINT NOT NULL
       - created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - deleted TINYINT NOT NULL DEFAULT 0
       - UNIQUE KEY uk_friendship (user_id, friend_id)
       - INDEX idx_friends (friend_id)

    6. **friend_requests** — 好友申请表
       - id BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY
       - from_uid BIGINT NOT NULL
       - to_uid BIGINT NOT NULL
       - status TINYINT NOT NULL DEFAULT 0（0=pending, 1=accepted, 2=rejected）
       - created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3)
       - updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
       - INDEX idx_pending_requests (to_uid, status)

    **DDL 约定（来自 PATTERNS.md 第 122-128 行）：**
    - 使用 `CREATE TABLE IF NOT EXISTS` 确保幂等
    - DATETIME(3) 精确到毫秒
    - 不写 FOREIGN KEY 约束（JPA 在应用层维护关系）
    - 索引命名规范：UNIQUE KEY uk_xxx、INDEX idx_xxx
    - 末尾空行
