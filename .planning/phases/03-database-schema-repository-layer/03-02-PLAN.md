    **MessageQueueRepository**（D-04, D-05）：
    - 构造参数：connection: StatefulRedisConnection<String, String>
    - 私有属性：val redis = connection.coroutines()
    - companion object：
      - const STREAM_KEY = "queue:messages"
      - const CONSUMER_GROUP = "flush-workers"
      - const CONSUMER_NAME = "worker-1"
    - **suspend fun ensureConsumerGroup()**：
      使用 `redis.xgroupCreate(STREAM_KEY, CONSUMER_GROUP, XReadArgs.StreamOffset.from(STREAM_KEY, "0-0"))`
      catch `RedisCommandExecutionException` 含 "BUSYGROUP" 信息时忽略（组已存在）
    - **suspend fun enqueue(message: Map<String, String>): String?**：
      `redis.xadd(STREAM_KEY, XAddArgs.Builder.maxlen(100000).approximateTrimming(), message)`
    - **suspend fun consume(batchSize: Long, blockMs: Long): List<StreamMessage<String, String>>**：
      `redis.xreadgroup(Consumer.from(CONSUMER_GROUP, CONSUMER_NAME), XReadArgs.StreamOffset.lastConsumed(STREAM_KEY), XReadArgs.Builder.count(batchSize).block(blockMs)) ?: emptyList()`
    - **suspend fun acknowledge(messageId: String)**：
      `redis.xack(STREAM_KEY, CONSUMER_GROUP, messageId)`
    - KDoc 引用 D-04, D-11
    - 添加 import：`io.lettuce.core.*`, `io.lettuce.core.api.coroutines.RedisCoroutinesCommands`

    - 使用 connection.coroutines() 获取协程命令接口（D-03）
