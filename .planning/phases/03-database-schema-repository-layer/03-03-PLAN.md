    创建目录 `repository/src/main/kotlin/com/nebula/repository/repository/impl/`，写入 MessageRepositoryImpl.kt。

    包名：`com.nebula.repository.repository.impl`

    **类**：`class MessageRepositoryImpl` implements `MessageWriteRepository`（D-08 第 7 个 Repository）
    - 构造参数：messageQueue: MessageQueueRepository, jpaMessageRepo: MessageRepository, emf: EntityManagerFactory, snowflakeIdGenerator: SnowflakeIdGenerator
    - `private val logger = KotlinLogging.logger {}`
    - `private var stopped = false`

    **方法列表：**

    1. **override suspend fun enqueueMessage(entity: MessageEntity): String**
       - 使用 SnowflakeIdGenerator 生成消息 ID：`entity.id = snowflakeIdGenerator.nextId()`
       - 设置 server_ts：`entity.serverTs = System.currentTimeMillis()`
       - 将 entity 序列化为 Map<String, String>（字段名→值映射），排除 null 值
       - 调用 `messageQueue.enqueue(map)` 写入 Redis Stream
       - 返回 Redis Stream 返回的消息 ID

    2. **override suspend fun flushBatch(): Int**
       - 调用 `messageQueue.consume(batchSize = 30, blockMs = 0)` 获取待刷写消息
       - 如果 entries.isEmpty() 返回 0
       - 将 StreamMessage 解析为 MessageEntity 列表
       - **解析方法**：从 StreamMessage.body（Map<String, String>）中提取字段重建 MessageEntity
       - 如果 messages.size < 30 返回 0（不到阈值不刷盘，但记录日志）
       - 使用 EntityManager 执行批量 INSERT：
         - `emf.createEntityManager()` → begin transaction
         - 循环 em.persist(msg)，每 30 条 flush+clear
         - commit transaction
       - 对每一条成功写入的消息调用 `messageQueue.acknowledge(it.id)`
       - 返回写入条数
       - catch Exception：logger.error，消息保留在 Redis Stream 重试

    3. **override suspend fun acknowledgeMessage(messageId: String)**
       - 委托给 messageQueue.acknowledge(messageId)
       - 供外部在消息已成功持久化后主动确认

    4. **fun startFlushTimer()**
       - 使用 `CoroutineScope(Dispatchers.IO).launch { while (!stopped) { delay(500); flushBatch() } }`
       - D-11：500ms 间隔，30 条阈值

    **关键实现细节**：
    - import `kotlinx.coroutines.*`
    - import `jakarta.persistence.EntityManagerFactory`
    - import `io.github.oshai.kotlinlogging.KotlinLogging`
    - 序列化/反序列化使用手动字段映射（简单 Map<String, String>，不引入 JSON 依赖）
    - KDoc 引用 D-03, D-04, D-10, D-11

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application → Redis Stream | Message data written to Redis before MySQL; intermediate data could be lost on crash |
| Flush worker → MySQL | Batch INSERT is transactional; partial failure could leave Redis/MySQL inconsistent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | MessageRepositoryImpl.parseToEntity | mitigate | StreamMessage body map values are parsed with null-safe access; invalid entries are skipped with a warning log |
| T-03-10 | Denial of Service | Flush timer CoroutineScope | mitigate | CoroutineScope(Dispatchers.IO) limits to platform default parallelism; scope is never exposed externally |
| T-03-11 | Spoofing | docker-compose.yml default credentials | accept | root/root123 and no Redis auth are for local dev only; production credentials configured via environment variables |
| T-03-12 | Information Disclosure | Redis Stream pending entries (PEL) | accept | Messages remain in PEL until XACK'd; if flush worker crashes, messages stay in PEL and are re-dispatched on restart via XAUTOCLAIM (Phase 10) |
</threat_model>
