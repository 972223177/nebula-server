    完整 proto/build.gradle.kts 内容：

    ```kotlin
    import com.google.protobuf.gradle.*

    plugins {
        id("org.jetbrains.kotlin.jvm")
        id("com.google.protobuf")
    }

    dependencies {
        implementation(libs.protobuf.java)
        implementation(libs.protobuf.kotlin)
        implementation("javax.annotation:javax.annotation-api:1.3.2")
    }

    protobuf {
        protoc {
            artifact = libs.protoc.get().toString()
        }
        generateProtoTasks {
            all().configureEach {
                builtins {
                    create("java") { }
                }
            }
        }
    }

    kotlin {
        jvmToolchain(21)
    }

    java {
        sourceSets {
            main {
                java.srcDir("build/generated/source/proto/main/java")
            }
        }
    }
    ```

    注意：
    - 使用 `create("java") { }` 语法（Pitfall 1 预防 — Kotlin DSL 不能直接写 `java { }`）
    - 不设置 `generatedFilesBaseDir`（Pitfall 2 预防 — 0.10.0 只读）
    - protoc artifact 通过 libs.versions.toml 的 libs.protoc 引用（D-13 版本统一管理）
    - 不需要配置额外的 Kotlin builtin — protobuf-kotlin 依赖自动提供 Kotlin 扩展（D-07 双生成）
    - jvmToolchain(21) 保持与其他模块一致
    - javax.annotation-api 依赖解决 Java 9+ 的 @Generated 注解缺失问题
