    - KDoc 注明 5 个字段的用途（userId/用户ID、token/会话Token、deviceType/设备类型、deviceId/设备标识、connectionId/gRPC连接标识）

    **SessionRegistry.kt** (D-17, D-18, D-19, D-20):
    - `class SessionRegistry(private val sessionRepository: SessionRepository)` — D-17
    - private 字段：
      - `localCache = ConcurrentHashMap<String, Session>()` — token → Session（L1）
      - `userIdIndex = ConcurrentHashMap<Long, MutableSet<String>>()` — userId → tokens（用于多设备管理，D-18）
      - `evictionCallbacks = CopyOnWriteArrayList<(String) -> Unit>()` — 回调列表（D-20）
    - `fun onEviction(callback: (token: String) -> Unit)` — 注册驱逐回调（D-20）
    - 细粒度方法（D-19）：
      - `addToLocalCache(session: Session)` — 写入 L1 + 更新 userIdIndex
      - `removeFromLocalCache(token: String): Session?` — 移除 L1 + 更新 userIdIndex
      - `getFromLocalCache(token: String): Session?`
      - 对应 Redis 方法（suspend）：
        - `suspend fun saveToRedis(session: Session)` — 通过 sessionRepository.save() 将 token→连接信息的 JSON 写入 Redis
        - `suspend fun removeFromRedis(token: String)` — 通过 sessionRepository.delete() 删除
        - `suspend fun queryFromRedis(token: String): Session?` — 通过 sessionRepository.findByToken() 查询并反序列化为 Session
    - 组合方法（D-19）：
      - `suspend fun validate(token: String): Session?` — `getFromLocalCache(token) ?: queryFromRedis(token)?.also { addToLocalCache(it) }`
      - `suspend fun register(session: Session)` — `addToLocalCache(session); saveToRedis(session)`
      - `suspend fun unregister(token: String)` — `removeFromLocalCache(token); removeFromRedis(token); evictionCallbacks.forEach { it(token) }`
    - KDoc 注明 "L1(ConcurrentHashMap) + L2(Redis SessionRepository) 二级缓存（D-18）"

      - `unregister triggers eviction callbacks` — 注册回调后 unregister，验证回调被调用

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Tampering | SessionRegistry L1/L2 一致性 | mitigate | L1 只读缓存（validate 写入），L2 持久化（Redis TTL）。L1 故障不影响全局 Session（L2 兜底） |
| T-04-11 | Elevation of Privilege | PingHandler | mitigate | D-30: AuthInterceptor 跳过 system/ping，但 PingHandler 只返回固定 pong 响应，无业务逻辑暴露 |

| T-04-13 | Elevation of Privilege | SessionRegistry.onEviction | mitigate | 回调仅通知关闭 StreamObserver，不涉及认证或授权操作（D-20） |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | 本计划无包管理器安装操作 |
