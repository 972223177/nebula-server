    ```kotlin
    // 当前实现不验证请求者是否为会话成员，任何认证用户可以拉取任何会话的消息。

    ```

    handle() 实现:
    1. `val session = coroutineContext.requireSession()`
    2. **会话存在性检查**（REVIEW-MEDIUM-9）：
       - `val exists = conversationRepository.existsById(req.conversationId)`
       - `if (!exists) throw ConversationException(BizCode.CONV_NOT_FOUND, "会话不存在")`
       - 这区分了"会话不存在"和"会话无消息"两种情况
    3. `val cursor = req.cursor`
    4. `val limit = req.limit.coerceIn(1, 100)`（D-19: 默认 20，最大 100 — limit 默认值由 proto 决定）
    5. cursor 特殊处理（D-18, Pitfall 2）：
       - cursor=0（无约束，查最新消息）→ `val effectiveCursor = Long.MAX_VALUE`
       - cursor>0 → `val effectiveCursor = cursor`
       - 添加注释说明：`// Snowflake ID 始终 < Long.MAX_VALUE，所以 MAX_VALUE 等效于"无上界"`
    6. `messageRepository.findMessagesBackward(req.conversationId, effectiveCursor, Pageable.ofSize(limit))`
    7. `hasMore = messages.size >= limit`
    8. Entity→ChatMessage 映射（扩展函数 `MessageEntity.toChatMessage()`）：
       - msgId → this.id!!
       - conversationId → this.conversationId
       - senderUid → this.senderUid
       - receiverUid → 0（群聊和私聊场景下，历史消息的 receiver_uid 消息落库时未写入该字段，默认为 0）
       - **messageType → ChatContentType.forNumber(this.messageType) ?: ChatContentType.UNRECOGNIZED**（REVIEW-MEDIUM-11：forNumber 可能返回 null，使用 UNRECOGNIZED 保底）
       - content → this.content
       - payload → ByteString.copyFrom(this.payload ?: ByteArray(0))
       - clientTs → this.clientTs
       - serverTs → this.serverTs
    9. 返回 `PullMessagesResp.newBuilder().addAllMessages(messages.map { it.toChatMessage() }).setHasMore(hasMore).build()`
    10. direction 字段保留不动（D-20）

    添加 `companion object { private val logger = KotlinLogging.logger {} }`
    KDoc 中文注释，引用 D-17, D-18, D-19, D-20, D-21, REVIEW-HIGH-3, REVIEW-MEDIUM-9, REVIEW-MEDIUM-11。
