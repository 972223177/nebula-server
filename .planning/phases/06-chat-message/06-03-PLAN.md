    创建 PullMessagesHandler（D-17, D-18, D-19, D-20, D-21）：
    package: `com.nebula.gateway.handler.message`
    构造注入: `MessageRepository`
    实现: `Handler<PullMessagesReq, PullMessagesResp>`
    - `override val method: String = "message/pull"`

    handle() 实现:
    1. `val cursor = req.cursor`
    2. `val limit = req.limit.coerceIn(1, 100)`（D-19: 默认 20，最大 100 — limit 默认值由 proto 决定）
    3. cursor 特殊处理（D-18, Pitfall 2）：
       - cursor=0（无约束，查最新消息）→ `val effectiveCursor = Long.MAX_VALUE`
       - cursor>0 → `val effectiveCursor = cursor`
    4. `messageRepository.findMessagesBackward(req.conversationId, effectiveCursor, Pageable.ofSize(limit))`
    5. `hasMore = messages.size >= limit`
    6. Entity→ChatMessage 映射（扩展函数 `MessageEntity.toChatMessage()`）：
       - msgId → this.id!!
       - conversationId → this.conversationId
       - senderUid → this.senderUid
       - receiverUid → 0（群聊和私聊场景下，历史消息的 receiver_uid 消息落库时未写入该字段，默认为 0。D-22 新增字段不在 MessageEntity 中——暂不修改 entity，统一填 0）
       - messageType → ChatContentType.forNumber(this.messageType)
       - content → this.content
       - payload → ByteString.copyFrom(this.payload ?: ByteArray(0))
       - clientTs → this.clientTs
       - serverTs → this.serverTs
    7. 返回 `PullMessagesResp.newBuilder().addAllMessages(messages.map { it.toChatMessage() }).setHasMore(hasMore).build()`
    8. direction 字段保留不动（D-20）

    添加 `companion object { private val logger = KotlinLogging.logger {} }`
    KDoc 中文注释，引用 D-17, D-18, D-19, D-20, D-21。
  </action>

    handle() 实现:
    1. `val session = coroutineContext.requireSession()`
    2. 获取会话类型（D-27）：
       - `val conversation = conversationRepository.findById(req.conversationId).orElse(null)`
       - conversation == null → `throw ConversationException(BizCode.CONV_NOT_FOUND)` — 使用 ConversationException（已有领域异常）
       - `val isPrivate = conversation.type == 0` — 0 表示私聊（定义 PRIVATE_TYPE=0 常量）
    3. 更新已读进度（D-24 更新 last_read_message_id）：
       `conversationMemberRepository.updateReadReceipt(req.conversationId, session.userId, req.lastReadMsgId)`
    4. 重置 Redis 未读计数键（D-28）：
       `redis.del("conversation:${req.conversationId}:unread:${session.userId}")`
       （D-28 接受极低概率竞态：DEL 后新消息 INCR 覆盖，下次新消息自动修复）
    5. 私聊场景推送已读回执（D-23）：
       - 如果 isPrivate:
         - 获取私聊发送者 UID：通过 `conversationMemberRepository.findByConversationIdAndUserId(req.conversationId, userId)` 查找发送者，或通过业务逻辑——私聊只有两个成员，读者是当前 session 用户，发送者就是另一个成员
         - 查询逻辑：`val members = conversationMemberRepository.findByConversationId(req.conversationId)` → 取 reader_uid != session.userId 的成员
         - 如果 members 有且 reader_uid != session.userId，reader_uid 就是 senderUid
         - 构建 ReadReceiptPayload（D-24）：`ReadReceiptPayload.newBuilder().setConversationId(req.conversationId).setReaderUid(session.userId).setMsgId(req.lastReadMsgId).build()`
         - `pushService.pushReadReceipt(senderUid, payload)`（D-25 使用已有 proto）
       - 群聊：不推送
    6. 返回 `Response.newBuilder().setCode(BizCode.OK.code).setMethod("message/read").build()`

    添加 `companion object { private val logger = KotlinLogging.logger {} }`
