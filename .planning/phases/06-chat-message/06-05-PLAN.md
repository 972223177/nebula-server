# Plan 06-05: 解决 JPA EntityManager 阻塞问题（Phase 6 范围）

## Objective

解决 Phase 6（Chat & Message）所有 Handler 中 JPA 阻塞操作阻塞 Netty 事件循环线程的问题。所有 JPA 操作（`JpaRepository` 接口调用如 `findById`、`findByConversationIdAndUserId`、`findMessagesBackward` 等）需通过 `withContext(Dispatchers.IO)` 切换到 IO 线程池执行。

## Problem

当前架构中，Handler 的 `suspend fun handle()` 由 Dispatcher 在 Netty 事件循环线程上调用。当 Handler 调用 Spring Data JPA Repository 方法时，这些方法内部通过 EntityManager 执行数据库查询，会阻塞当前线程。在 Netty 事件循环线程上执行这些操作会阻塞所有 gRPC 请求处理。
