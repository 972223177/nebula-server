    **ValidateStep.kt**（D-08, D-13, D-14）：
    package: `com.nebula.gateway.handler.chat.send`
    构造注入: `ConversationMemberRepository`
    execute():
    1. req.content.isBlank() → throw SendMessageException(BizCode.INVALID_PARAM, "消息内容不能为空")（D-14）
    2. req.clientMessageId.isBlank() → throw SendMessageException(BizCode.INVALID_PARAM, "client_message_id 不能为空")（D-14）

    **DedupStep.kt**（D-07, D-13）：
    package: `com.nebula.gateway.handler.chat.send`
    构造注入: `StatefulRedisConnection<String, String>`
    使用 `@OptIn(ExperimentalLettuceCoroutinesApi::class)` 和 `RedisCoroutinesCommandsImpl(connection.reactive())`
    execute():
    1. `val dedupKey = "chat:dedup:${context.req.clientMessageId}"`
    2. `val existed = redis.setex(dedupKey, 7 * 24 * 3600L, "pending")` — Redis SETNX 语义：键不存在时 set 并返回 null（成功），键已存在时返回当前值（重复）
    3. 如果 existed != null（重复消息）→ `throw SendMessageException(BizCode.SEND_FAILED, "重复消息")`（D-07 忽略重复）
    4. return true
    注意：`setex` 的返回值语义取决于 Lettuce coroutines 实现。如果 `setex` 不返回 null 表示"键已存在"，改用 `setnx`（Redis SETNX 命令）：
    ```kotlin
    val set = redis.setnx(dedupKey, "pending")  // true=set成功, false=键已存在
    if (!set) throw SendMessageException(...)
    redis.expire(dedupKey, 7 * 24 * 3600L)
    ```
    使用 setnx + expire 的组合确保去重语义正确。
    KDoc 引用 D-07, D-13。

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | ValidateStep | mitigate | content 非空和 client_message_id 非空校验（D-14）；成员身份 SISMEMBER 查询（D-08） |
| T-06-05 | Spoofing | DedupStep | mitigate | Redis SETNX 是原子操作（D-07），7 天 TTL 限制重放窗口；客户端必须有唯一 client_message_id |
| T-06-06 | Elevation of Privilege | WriteStep | mitigate | 仅写入已验证成员发起的消息（ValidateStep 已前置检查成员身份） |
| T-06-07 | Denial of Service | PushStep | mitigate | 每个 receiver 独立 try-catch（D-05）；单个推送异常不阻断其他推送（D-12） |
</threat_model>
