    ```kotlin
    import io.grpc.netty.GrpcSslContexts
    import io.netty.handler.ssl.SslContext
    import io.netty.handler.ssl.SslProvider
    import java.io.File
    import java.security.Security

    fun SslConfig.buildSslContext(): SslContext? {
        if (!enabled) return null
        val certChain = File(certChainPath)
        val privateKey = File(privateKeyPath)
        return GrpcSslContexts
            .forServer(certChain, privateKey)
            .sslProvider(SslProvider.OPENSSL)     // 性能最优，需 netty-tcnative classpath
            .protocols("TLSv1.2", "TLSv1.3")
            .ciphers(
                listOf(
                    "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
                    "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384"
                ),
                Security.getProvider("SunJSSE")
            )
            .build()
    }
    ```

    注意：SslProvider.OPENSSL 需要 netty-tcnative classpath（已在 Plan 01 的 server/build.gradle.kts 中添加依赖）。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 配置文件 → ConfigLoader | 文件系统读取，非用户输入，但可能被篡改（git 追踪） |
| gRPC 客户端 → NettyServer | 不受信的外部输入，网络边界（per V10 Communication Security） |
| 数据库连接 → MySQL | JDBC 连接，凭据通过环境变量注入 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Denial of Service | NettyServerBuilder | mitigate | `permitKeepAliveWithoutCalls(false)` 防止空连接消耗 keepalive 资源；`maxInboundMessageSize(4MB)` 限制消息大小（per INFRA-02 + 安全需求） |
| T-02-08 | Information Disclosure | SSL SslContext | mitigate | 生产环境使用 Let's Encrypt 证书（路径通过环境变量配置 per D-10）；密码套件限制为 ECDHE + AES-GCM（PERF RESEARCH.md Pattern 2） |
| T-02-09 | Tampering | HikariDataSourceProvider | mitigate | 数据库凭据通过 `${DB_PASSWORD}` 环境变量注入（per D-03），不在文件或代码中硬编码 |
| T-02-10 | Information Disclosure | JDBC SSL | mitigate | 使用 mysql-connector-j 9.x 兼容的 `sslMode=PREFERRED`（开发）或 `sslMode=VERIFY_IDENTITY`（生产，通过修改 application.conf 中的 JDBC URL 模板实现），替代已废弃的 `useSSL` 参数（per Pitfall 2） |
| T-02-11 | Denial of Service | HikariCP leakDetection | mitigate | `leakDetectionThreshold=10000ms` 启用泄漏检测（per 设计文档 9.5），HikariCP 在连接未归还超时后自动打印警告 |
| T-02-12 | Spoofing | ConfigFactory.parseFile | accept | 配置文件提交 git 版本控制（per D-03），单节点部署场景下替换配置需要文件系统写权限 |
</threat_model>
