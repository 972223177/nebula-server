# Plan 11-01：安全与生产加固

## 目标

修复 13 个 HIGH 严重性安全漏洞和基础设施缺陷，包括：密钥明文存储、SSL/TLS 弱模式、缺少 Shutdown Hook、无登录审计日志、RateLimiter 内存泄漏等，使服务具备生产环境安全基线。

## 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 移除 `DB_PASSWORD` 默认值导致开发环境启动失败 | 开发效率降低 | 提供 `.env.example` 模板 + README 说明；CI 中设置环境变量 |
| `sslMode=VERIFY_CA` 需配置 CA 证书 | 首次部署复杂度增加 | 文档说明 CA 证书导入步骤；内网环境可用 `REQUIRED` 降级 |
| Shutdown Hook 中清理步骤可能耗时过长 | JVM 强杀导致资源未释放 | 每个清理步骤 ≤ 5 秒；`kill -9` 场景无法防御（JVM 层面限制） |
| RateLimiter 定时清理误删活跃用户 | 用户被错误解除限流 | 使用 `availablePermits == permitsPerUser` 条件：仅在所有许可都可用（用户完全无请求）时移除 |
