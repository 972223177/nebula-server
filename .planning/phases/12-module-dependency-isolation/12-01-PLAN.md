# 12-01: 模块依赖隔离 — 全部 api → implementation 并消除跨层引用

## 目标

将 `repository/build.gradle.kts` 的 `api(project(":common"))` 和 `service/build.gradle.kts` 的 `api(project(":repository"))`, `api(project(":proto"))` 改为 `implementation`，并消除由此暴露的所有编译时跨层引用。

## 执行上下文

执行方式：直接编码（跳过 discuss 阶段，工作已通过 agent 团队完成）

## 执行策略

采用 5 个任务组，其中任务组 1 必须先完成（基础抽象层），任务组 2-4 并行执行，任务组 5 最后执行。

### 任务组 1: 新增 service 层抽象（基础层，先执行）

| # | 任务 | 产出 |
|---|------|------|
| 1.1 | 创建 OnlineStatusService 包装 OnlineStatusRepository | `service/.../user/OnlineStatusService.kt` |
| 1.2 | 扩展 UserPrivacyService: batchGetHideOnlineStatus() | `service/.../user/UserPrivacyService.kt` |
| 1.3 | 创建 SessionStore 接口 (common 模块) | `common/.../session/SessionStore.kt` |

### 任务组 2: Handler 重构（并行）

修改 10 个 Handler 文件，将 repository 注入改为 service 注入:

- BatchGetStatusHandler: OnlineStatusRepository → OnlineStatusService + UserPrivacyService
- SetPrivacyHandler: OnlineStatusRepository + FriendshipRepository → OnlineStatusService + FriendService
- SendMessageHandler: ConversationMemberRepository + MessageQueueRepository → ConversationService + MessageService
- ReadReportHandler: ConversationRepository + ConversationMemberRepository → ConversationService
- FriendAddHandler: FriendshipRepository → FriendService
- FriendAcceptHandler: 删除未使用的 FriendshipRepository 参数
- LeaveGroupHandler: ConversationMemberRepository → ConversationService
- InviteMemberHandler: 删除未使用的 repository 参数
- KickMemberHandler: 删除未使用的 repository 参数
- DeadLetterQueryHandler: DeadLetterEntity → DeadLetterDTO

### 任务组 3: Gateway 核心组件重构（并行）

- ChatService: onlineStatusRepo/friendshipRepo/privacyRepo → OnlineStatusService/FriendService/UserPrivacyService
- PushService: conversationMemberRepository → ConversationService
- SessionRegistry: SessionRepository → SessionStore

### 任务组 4: ServerBootstrap 重写 + DI 模块更新（最后）

- ServerBootstrap.kt: koinModules → ServiceInitModule; recoverSequences → SeqService; createChatService → Koin.get(); 死信桥接 → DeadLetterCallback
- FrameworkModule: 注册 Dispatcher + ChatService
- GatewayModule: serviceModule → serviceKoinModule (从 service 导入)
- ServiceModule: 移至 service 模块，删除 gateway 中的旧版本
- MessageReliabilityModule: 移除 DeadLetterService/SeqService 构造（已在 serviceKoinModule）
