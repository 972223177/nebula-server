# Plan 05-05: 解决 JPA EntityManager 阻塞问题（Phase 5 范围）

## Objective

解决 Phase 5（User & Authentication）所有 Handler 中 JPA 阻塞操作阻塞 Netty 事件循环线程的问题。所有 JPA 操作（`EntityManager`、`JpaRepository` 查询）需通过 `withContext(Dispatchers.IO)` 切换到 IO 线程池执行，避免阻塞 Netty 事件循环。

## Problem

当前架构中，Handler 的 `suspend fun handle()` 由 Dispatcher 在 Netty 事件循环线程上调用。当 Handler 直接调用 JPA 阻塞操作（如 `userRepository.findById()`、`emf.createEntityManager()`）时，Netty 事件循环线程被阻塞，导致所有 gRPC 请求被挂起。
